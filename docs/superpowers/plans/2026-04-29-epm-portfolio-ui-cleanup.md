# EPM Portfolio UI Cleanup

Date: 2026-04-29
Branch: `improvement/epm-portfolio-ui-cleanup`

## Problem

In EPM "All projects" view:

1. The `filters-strip` block ("PROJECT — All visible Atlassian Home projects.") duplicates the project picker above and adds visual noise.
2. The "N issues appear in multiple projects." line is uninformative — does not tell the user which issues, which projects, or how to fix it.
3. Project group headers (`group-pane-title`, 0.7rem, secondary text) are nearly invisible. Users do not register that there are distinct project sections.
4. The settings (gear) button on the EPM screen lands on the first settings tab (Scope) instead of EPM → Projects.

## Changes

### E. Settings tab reset bug

`frontend/src/dashboard.jsx:1410` — delete `setGroupManageTab('scope');` from the open-effect (`useEffect` with deps `[showGroupManage, groupsConfig]`).

The opener (`openEpmSettingsTab`) already sets the tab; this line was wiping it. The close handler (`closeGroupManage`) still resets to `'scope'` on close, so subsequent opens without an explicit tab choice still default to Scope.

No other call site relies on the open-effect to set the tab.

### B. Remove EPM filters-strip

`frontend/src/dashboard.jsx:15076-15090` — delete the `<div className="filters-strip">…</div>` block. The selected-project status info (update line) moves into the new project header.

### D. Project header + collapse

`frontend/src/epm/EpmRollupPanel.jsx` — replace the `<div className="group-pane-title">` heading inside `epm-project-board` with a new initiative-style header.

Header markup:

- Project icon (small SVG, similar visual weight to ENG `InitiativeIcon`).
- Project name in larger, primary text (matches ENG `initiative-label-name`).
- Jira label as a small mono pill (e.g. `rnd_project_data_partnership`).
- "Open in Jira Home" link (when `project.homeUrl` is set).
- Update line (small secondary text under name) when present.
- Chevron button toggling collapse.

Body markup:

- Wrapped in `epm-project-board-body` with the same vertical accent line treatment as ENG `initiative-body::before`.
- Hidden when collapsed (`display: none` via class).

State:

- `collapsedProjectIds: Set<string>` in component, default empty (all expanded).
- Toggle on chevron click; whole header is the click target.
- Session-only — not persisted.
- No bulk Collapse-all / Expand-all control in this pass.

CSS additions (in `frontend/dist/dashboard.css`):

The `metadata-only` and `emptyRollup` rendering inside the section continue to render unchanged inside the body.

### C. Duplicates callout

`frontend/src/epm/EpmRollupPanel.jsx` — replace the line at lines 105-109 with a structured callout.

Data shape: `epmDuplicates = { issueKey: [projectId, ...] }`.

Build clusters:

```
clusters = group_by(epmDuplicates, value => sortedJoin(projectIds))
```

For each cluster, pull project records from `epmRollupBoards` to get name + label + Jira home URL.

Render:

- Heading: "Issues counted in multiple projects".
- One row per cluster:
  - Project chips: `<name> · <label-pill>` for each project in the cluster.
  - "N issues" with a disclosure (`<details>`) listing keys linked to `${jiraUrl}/browse/${key}`.
  - "Fix labels" button → `openEpmSettingsTab` (opens EPM Settings → Projects; precise targeting deferred).

CSS additions:

- `.epm-duplicates-callout` — bordered card, warm accent.
- `.epm-duplicates-row` — flex row, gap, divider between rows.
- `.epm-duplicates-projects` — wrap of project chips.
- `.epm-duplicates-project-chip` — name + mono pill.
- `.epm-duplicates-issues` — disclosure styling.
- `.epm-duplicates-fix-button` — primary-ish action.
