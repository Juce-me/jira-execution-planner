QUICK START - Current Local Setup
=================================

For the supported DB/OAuth path, use INSTALL.md. This file is a short checklist.

1. INSTALL DEPENDENCIES IN A PROJECT VIRTUALENV

   ./install.sh

   Or manually:

   python3 -m venv .venv
   . .venv/bin/activate
   python -m pip install -r requirements.txt
   python -m pip install -e .

2. CREATE .env

   cp .env.example .env

3. CONFIGURE DB/OAUTH VALUES

   Set the values documented in INSTALL.md:

   - JIRA_URL
   - JIRA_AUTH_MODE=atlassian_oauth
   - Atlassian OAuth client settings
   - CONFIG_STORAGE_BACKEND=db
   - DATABASE_URL
   - TOKEN_ENCRYPTION_MASTER_KEY_B64
   - TOKEN_ENCRYPTION_KEY_ID
   - FLASK_SECRET_KEY

4. RUN DATABASE MIGRATIONS

   export DATABASE_URL=postgresql+psycopg:///jep_local
   .venv/bin/alembic -c backend/db/alembic.ini upgrade head

5. CHECK STARTUP PREREQUISITES

   .venv/bin/python scripts/check_startup_preflight.py

6. START SERVER

   .venv/bin/python jira_server.py

7. OPEN DASHBOARD

   http://localhost:5050/

8. CONNECT HOME TOKEN FOR EPM

   Sign in, then open Settings -> Connections and connect the signed-in user's
   Home/Townsquare token. DB/OAuth EPM stays hidden until that encrypted user
   token exists.

PREBUILT RELEASE ZIP
====================

Release zips are the runnable package for non-development use. They include
frontend/dist and omit frontend source tests/plans. Unzip the release, run
./install.sh, configure .env, run migrations for DB/OAuth mode, then run the
startup preflight before launching the server.

LEGACY BASIC MODE
=================

Server-side API-token auth remains only a local compatibility mode. Do not use
legacy Jira/Home Basic credentials for the DB/OAuth EPM path.
